# 碎片化度量（图谱健康度）

本文档钉死知识图谱"碎片化"的**定义、度量口径、健康阈值与处置原则**。lint 100 分只保证格式合规，不保证结构健康；碎片化是结构层问题，必须用图拓扑口径度量，不能用 lint 分数替代。

## 为什么单独成档

历史上出现过 agenda 写"81.8% entity 零入链"而实测为 1.1% 的基线漂移：同一份图谱，用不同"连接"定义会得出差一个数量级的结论，并驱动量级错配的下一步动作。根因是**口径没有被 schema 钉死、数字没有可复现工具**。本文档是口径的单一来源。

## 连接口径三档（防漂移的根本）

讨论任何"碎片化/孤岛/连接度"前，必须先声明用哪一档口径。三档互不等价，混用即误诊。

| 口径 | 定义 | 现有工具字段 | 适合诊断 |
|------|------|-------------|---------|
| **全图入链** | 被 `wiki/` 下任意 .md 引用 | `entity-audit.py` 的 `graph_in` | 绝对孤岛（完全无人引用） |
| **知识层互引** | 被其他 entity 引用 | `entity-audit.py` 的 `entity_in` | entity↔entity 拓扑健康 |
| **整合层承载** | 被 topic/comparison 引用 | `entity-audit.py` 的 `topic_in`+`comparison_in` | 整合层覆盖是否充分 |

> `source`（证据层）对 entity 的引用只算"证据连接"，不算"知识整合"。一个 entity 被多篇 source 引用但无 entity 互引、无 topic 承载，属于"有证据无知识整合"，是整合层稀薄的典型形态，不是绝对孤岛。

## 五层碎片化

### 第 1 层 · 整合层稀薄

- **定义**：entity 存在且有互引，但未被任何 topic/comparison 承载，散在互引网上未组织进可读主题叙事。
- **口径**：整合层承载口径（被 topic/comparison 引用）。
- **阈值**：未承载 entity 占比 ≤ 20% 为健康；> 35% 为碎片化。
- **测量**：`entity_in==0 and topic_in==0 and comparison_in==0` 但 `graph_in>0` 的 entity 集合；按 primary tag 聚类看是否能成簇建 topic。
- **处置**：对未承载 entity 聚类，成簇的建 topic 收拢；不成簇的并入相邻 topic 小节或降级。**优先补整合层，不优先补 entity 互引**——entity 互引已健康时，加互引是无效劳动。

### 第 2 层 · 幂律集中

- **定义**：少数 hub entity 吃掉大部分连接边。
- **口径**：知识层互引口径。
- **阈值**：**默认不设强制阈值，倾向不干预**。hub 集中是知识库的自然形态，Agentic-Engineering 这类超级 hub 是健康导航节点，强行打散破坏可导航性。
- **测量**：top-N hub 占总边数比例；中间层（入度 4–30）entity 占比。
- **真正要监控的子指标**：**单点挂靠**——仅被 1 个 entity 引用的 entity 占比。单点挂靠 entity 挂在一根线上，hub 变动即掉链。目标 < 10%。

### 第 3 层 · 语义 tag 碎片

- **定义**：tag 作为跨页面语义编织层失效——一次性 tag（全库仅出现 1 次）堆积、同概念域被切成多个 tag。
- **口径**：tag 维度，非图拓扑。
- **阈值**：一次性 tag 占比 < 15% 为健康。
- **测量**：lint 报告的 `tag` 检查项（一次性 tag 明细）；人工识别同概念域多 tag（如 `ai-cost`/`AI-economy`/`AGI-economics`/`ai-analysts`）。
- **处置**：收敛为受控词表，同概念域多 tag 合并；一次性 tag 评估后删除或并入高频 tag。**这是被 lint PASS 掩盖的语义碎片**——lint 只警告不阻断，需主动治理。

### 第 4 层 · 产出断层

- **定义**：research 层活跃但 synthesized 判断停在 research-logs，既不升级为 entity/topic，也不产出为 output。
- **口径**：research-logs 思考数 → output 数 的转化率。
- **阈值**：转化率 ≥ 20% 为健康；< 10% 为断层。
- **测量**：research-logs 中 synthesized 判断条数 / output 数；最后 output 距今天数。
- **处置**：建 research→output 标准转化路径；从 research-logs 提取成熟判断按 output-workflow 回填门控升级。**属流程层碎片，与图拓扑无关，但同等重要。**

### 第 5 层 · 基线漂移

- **定义**：agenda / 盘点文档中的碎片化数字与实测脱节，且无工具可复现。
- **口径**：元层——度量本身的可靠性。
- **阈值**：**零容忍**。agenda 中任何碎片化/连接度/孤岛率数字必须能由 `entity-audit.py` 或等价脚本复现，并标注测量口径与日期。
- **测量**：定期用 entity-audit.py 跑全库，与 agenda 数字比对。
- **处置**：发现漂移立即纠正，标注口径与日期；删除基于错误基线驱动的下一步动作。

## 基线快照（2026-06-25）

> 后续度量以此为对比基准。复现命令：`python3 tools/entity-audit.py`。

| 指标 | 口径 | 实测 | 健康判定 |
|------|------|------|---------|
| 全图零入链 entity | 全图入链 | 1.1% (3/280) | 健康 |
| 知识层零互引 entity | 知识层互引 | 6.8% (19/280) | 健康 |
| 整合层未承载 entity | 整合层承载 | 35.0% (98/280) | **碎片化** |
| 被 topic/comparison 引用 | 整合层承载 | 65.0% (182/280) | — |
| 单点挂靠（仅 1 互引） | 知识层互引 | 16.8% (47/280) | 偏高（目标 <10%） |
| Top 12 hub 占边数 | 知识层互引 | 22.0% | 温和，不干预 |
| 中间层（入度 4–30） | 知识层互引 | 50.7% | 骨干健康 |
| 一次性 tag | tag 维度 | 78 条 | 偏高，需收敛 |
| output 转化率 | 产出层 | 5 output / 67+ 思考 ≈ 7% | **断层** |

**结论**：碎片化真实形态是**第 1 层整合层稀薄 + 第 4 层产出断层**为主，第 3 层 tag 碎片为辅，第 2 层幂律不干预，第 5 层基线已纠正（见下）。不是"孤岛太多"。

## 防漂移规则

1. agenda / 盘点 / output 中引用任何碎片化数字，必须标注**口径档位 + 测量日期**，且能由 entity-audit.py 复现。
2. 不可复现的数字（手算、一次性脚本产物）不得写入 agenda 的"当前研究焦点"或"活跃假设"驱动优先级。
3. 全库盘点时必须先跑 entity-audit.py 生成基线快照，再与 agenda 比对，发现漂移当场纠正。
4. 本档基线快照表随每次全库盘点更新；旧快照保留版本以追踪漂移历史。
