# Observability for developers building connectors

Developers can now monitor their connectors' performance across Claude products and submit connectors to the directory in-app.

**Monitor, debug, and improve connectors**
Published connectors in the directory now have a dashboard showing:
*   **Adoption metrics:** Active users, total tool calls, and directory rank.
*   **Health signals:** Error rates and latency with per-tool breakdowns.
*   **Usage breakdown:** Engagement across Claude, Claude Code, and Cowork.

**Joining the directory**
Connectors are built on the Model Context Protocol (MCP). There are over 300 third-party connectors in the directory. Developers can now submit their MCP servers directly within the Claude interface under Organization settings.
