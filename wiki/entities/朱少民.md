# 朱少民

> [!definition] 定义
> **朱少民** 是软件工程领域作者，已出版 20+ 本书籍。他的核心贡献是揭示了 AI Agent 时代 Skills 沉淀中的委托-代理困境，并提出了制度创新而非技术方案的破局路径。

|------|------|

| 持续 | 年稿税收入十几万，为创作提供经济保障 |

## 核心观点

### Skills 蒸馏的委托-代理困境

朱少民的核心洞察是：Skills 沉淀制造了一个经典的委托-代理困境。企业希望工程师把隐性知识蒸馏成可复用的 Skills，但工程师清楚这等于训练自己的替代品。理性博弈导致三种对抗策略：

| 对抗策略 | 表现 |
|---------|------|
| 消极抵抗 | 不主动分享、拖延配合 |
| 质量投毒 | 故意写入低质量或有缺陷的 Skills |
| 上下文污染 | 提供不完整的上下文，让 Skills 在特定条件下失败 |

### 制度破局五招

朱少民提出五个制度方案来改写激励结构：

| 方案 | 机制 | 解决的问题 |
|------|------|-----------|
| Skills 版税制 | 持续分润 | 让知识沉淀成为持续收入来源 |
| 角色跃迁制 | 创建 Skills 是晋升阶梯 | 让知识贡献成为职业上升路径 |
| 对抗性蒸馏 | 系统自动观察 + 工程师审核 | 降低主动配合的门槛 |
| Skills 竞技场 | 同侪声望 | 让高质量 Skills 获得认可 |
| 组织保险 | 保护期、转岗优先权 | 提供根本安全感 |

### 技术纵深防投毒

朱少民强调激励不够，还必须有技术纵深来防止投毒：

- 形式化验证
- 血缘追踪（追踪 Skill 的来源和修改历史）
- 对抗性测试
- 灰度发布
- 多源交叉验证
