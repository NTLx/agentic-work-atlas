# Osmosis Learning（渗透式学习）

> [!definition] 定义
> **Osmosis Learning（渗透式学习）** 是一种非正式学习机制，类似物理学中的渗透现象：知识自发从高浓度区域（专家、高效实践）向低浓度区域（初学者、普通员工）流动。它不依赖正式培训，而是通过观察他人的工作过程自然习得。

## 关键数据点

- 它是 Shopify 核心价值观”Constant Learner”的技术实现方式。
- 通过 River Agent 的公开频道，员工可以在不打扰专家的情况下”回放”专家的思考路径。
- 传统渗透学习发生在办公室，通过听别人打电话或站在别人屏幕后；在远程办公和 AI 时代，这种学习几乎消失——需要通过制度设计重新激活。

## 两个核心前提

1. **工作可见**：所有人的工作过程必须最大程度透明。
2. **数字近距离**：虽然是远程/线上工作，但通过 Slack 频道等公共空间创造了”数字近距离”。

## AI 时代渗透学习的重新激活

传统渗透学习依赖物理空间的”偶然观察”——在办公室走过同事屏幕、听到专家打电话。AI 和远程工作让这种”偶然观察”几乎消失。Shopify 通过 [[Lehrwerkstatt|教学工坊]] 和 [[Public-only-Constraint|公开频道约束]] 重新创造了数字时代的渗透学习：

| 传统渗透学习 | AI 时代渗透学习 |
|-------------|---------------|
| 听到专家打电话 | 看到专家如何写 Prompt |
| 站在专家屏幕后 | 回放专家与 Agent 的对话 |
| 偶然听到讨论 | 搜索公开频道的历史对话 |
| 学习操作技巧 | 学习判断力和品味（[[Taste]]） |

关键转变是：AI 时代的渗透学习不再只是学习”怎么做”（操作技巧），更重要的是学习”为什么这么做”（判断力）和”为什么不那样做”（品味）。看到专家如何在 Agent 给出的三个方案中通过品味做出选择，是传统渗透学习无法提供的深度学习。

## 前提与局限性

- **前提**：必须消除”向工具求助是羞耻的”心理障碍。
- **局限**：无法替代深度的系统性学习（如算法基础、架构理论），更多是习得”默会知识（Tacit Knowledge）”和操作技巧。
- **可复制性**：需要强工程文化和高层明确背书；低信任、强层级组织可能难以直接迁移。
- **信噪比**：公开交互会产生噪音，需要搜索、标签和知识整理机制配合。
