# Osmosis Learning

> **核心洞察**：知识不是被“教会”的，而是被“熏会”的。

## 定义

**渗透式学习**（Osmosis Learning）描述了一种非正式的学习机制。它类似于物理学中的渗透现象：知识自发地从高浓度区域（专家、高效实践）向低浓度区域（初学者、普通员工）流动。

在 [[Tobi-Lütke]] 看来，这需要两个核心前提：
1. **工作可见**：所有人的工作过程必须最大程度透明。
2. **近距离**：虽然是远程/线上工作，但通过 Slack 频道等公共空间创造了“数字近距离”。

## AI 时代的特殊性

传统渗透学习发生在办公室，通过听别人打电话或站在别人屏幕后。在远程办公和 AI 时代，这种学习几乎消失。
Shopify 通过强制 Agent 公开化（[[Lehrwerkstatt]]），重新创造了数字时代的渗透学习：
- 看到专家如何通过 Prompt 挖掘 Legacy Code 的陈年 bug。
- 看到专家如何在 Agent 给出的三个方案中通过“品味（Taste）”做出选择。

---

## 关键数据点

- 它是 Shopify 核心价值观“Constant Learner”的技术实现方式。
- 通过 River Agent 的公开频道，员工可以在不打扰专家的情况下，“回放”专家的思考路径。

## 前提与局限性

- **前提**：必须消除“向工具求助是羞耻的”这种心理障碍。
- **局限性**：无法替代深度的系统性学习（如算法基础、架构理论），更多是习得“默会知识（Tacit Knowledge）”和操作技巧。
