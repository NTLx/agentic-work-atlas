# How one Anthropic seller rebuilt his team's workflows with Claude Code

## 编译摘要

### 1. 浓缩
- **核心结论1**: 非技术人员可以通过 Claude Code 构建战略性工具，技术壁垒正在消失
  - 关键证据: Jared Sires 作为 GTM 产品经理，在加入 Anthropic 前从未写过代码，但使用 Claude Code 自动化了收件箱和销售工作流程
- **核心结论2**: Claude Code 可以构建完整的 GTM 工具套件，包括邮件草拟、日常简报和销售插件
  - 关键证据: CLAFTS（Claude Drafts）用于草拟客户邮件；Daily Brief & Daily Recap 用于会议准备和跟进；Sales Plugin 包含 20+ 技能和 MCP 连接器，被 80% 的 Anthropic 销售组织使用
- **核心结论3**: 角色从"产品经理"转变为"GTM 架构师"，强调技术壁垒的消融
  - 关键证据: Jared 的角色已转变为"GTM 架构师"，他设计产品并与高级工程师合作实施

### 2. 质疑
- **关于"技术壁垒消失"的质疑**: 这是否适用于所有非技术人员？需要哪些前提条件（如 AI 素养、领域知识）？
- **关于"80% 使用率"的质疑**: 这个数据是如何测量的？是主动使用还是被动集成？
- **关于"角色转变"的质疑**: 这种角色转变是否可持续？还是只是特定组织文化的产物？

### 3. 对标
- **跨域关联1**: 此现象类似"公民开发者"运动，AI 正在将编程能力民主化
- **跨域关联2**: GTM 架构师角色类似传统 IT 中的"业务分析师"，但技术能力边界更模糊
