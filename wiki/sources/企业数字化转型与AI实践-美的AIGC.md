# 企业数字化转型与AI实践——美的AIGC实践与Agent等级标准

## 编译摘要

### 1. 浓缩
- **核心结论1**: 美的建立了 A1-A6 智能体能力等级标准（参考 OpenAI 5 levels / 工信部），从问答助手到组织自治的六级演进路径
  - 关键证据: A1 问答助手→A2 工具执行→A3 单智能体→A4 智能体协作→A5 决策专家→A6 组织自治；11.4 万用户、日活 2.8 万、Token 日消耗 1000 亿
- **核心结论2**: 智能体工厂（Factory Agent）是企业级 agent 的组织形态——实时感知、自主决策、高效执行、动态优化、持续进化
  - 关键证据: 基于企业大量标准流程重构为标准化 Skill（1812 个 Skill、279 个 MCP）；26 年从"焦虑观望"转为"积极行动"
- **核心结论3**: AI Native 原生组织的三个待解问题——ROI 如何体现在经营指标、一人多岗的组织形态、Token 消耗的经济边界
  - 关键证据: "不是所有任务都用 Token 跑"——agent 需要区分判断/创意环节与脚本可完成的环节

### 2. 质疑
- **关于"A1-A6 等级"的普适性**: 等级标准是否适用于不同行业？制造业的"组织自治"与知识工作的"组织自治"含义可能截然不同
- **关于"Token 日消耗 1000 亿"**: 规模数据缺乏成本结构分解；1000 亿 Token 中多少用于 agent 推理、多少用于 RAG 检索、多少用于对话？
- **关于"标准化 Skill"**: 1812 个 Skill 的质量参差不齐可能性很高；标准化流程的 Skill 化是否真的比规则引擎更有效？

### 3. 对标
- **Agent-Harness**: 美的的智能体工厂是 [[Agent-Harness]] 在中国大型制造企业的实践——harness 不仅是技术框架，而是包含 Skill 资产、MCP 连接和组织流程的完整系统
- **AI-Factory**: 美的的 AIGC 实践是 [[AI-Factory]] 概念的中国制造业实例——从工具到劳动力的演化路径
- **Skill-System**: 1812 个 Skill 的资产管理与 [[Skill-Internalization]] 中 Skill 内化机制相关，但更偏组织级而非个人级
- **Organization-as-Agent-Harness**: 美的的"一人多岗、人+硅基中层"探索直接关联 [[Organization-as-Agent-Harness]] 中组织作为 harness 的概念
